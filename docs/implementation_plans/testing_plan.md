# Testing Plan

Generated by scanning the `job_search_tool` codebase after Prompts 21–24.

---

## 1. What to Test

### 1.1 `utils/resume_parser.py`
- **Functions:** `parse_resume()`, `preview_resume()`
- **Why:** PDF parsing is brittle (image-based PDFs, malformed files). A regression here breaks every command.

### 1.2 `utils/tracker.py`
- **Functions:** `add_application()`, `application_exists()`, `delete_application()`, `edit_application()`, `update_status()`, `get_followups_due()`, `show_tracker()`
- **Why:** Data integrity is critical. A bug could silently drop applications or corrupt the spreadsheet.

### 1.3 `chains/analyzer.py`
- **Functions:** `analyze_job()`, `validate_job_description()`
- **Why:** `validate_job_description` is pure logic and cheap to test. `analyze_job` requires LLM mocking but is the core value proposition.

### 1.4 `chains/tailorer.py`
- **Function:** `tailor_resume()`
- **Why:** Requires LLM mocking. Test that the prompt template is correctly populated with `must_have` and `matching_skills`.

### 1.5 `chains/cover_letter.py`
- **Function:** `generate_cover_letter()`
- **Why:** Requires LLM mocking. Test prompt variable population.

### 1.6 `chains/followup.py`
- **Function:** `draft_followup()`
- **Why:** Requires LLM mocking. Simplest chain — good for proving the mock setup works.

### 1.7 `config.py`
- **Function:** `validate_config()`
- **Why:** Should not print warnings when config is valid; should warn when Ollama port is closed or resume is missing.

### 1.8 `main.py` (CLI)
- **Commands:** `verify`, `analyze`, `apply`, `followup`, `tracker`, `status`
- **Why:** Integration tests using Typer's `CliRunner` to confirm argument parsing, option combinations, and exit codes.

### 1.9 `app.py` (Streamlit)
- **Why:** Streamlit is hard to unit-test. Recommend manual smoke tests rather than automated unit tests for UI layout.

## 3. Mocking Strategy

### LLM Calls
- **Recommended:** LangChain's `FakeListChatModel` for chain tests. It returns preset strings and integrates with LCEL pipes without network calls.
- **Alternative:** `unittest.mock.patch` on `chains.llm.get_llm` to return a mock `ChatOllama` whose `.invoke()` returns a preset result.
- **Sample setup for FakeListChatModel:**
  ```python
  from langchain_community.chat_models import FakeListChatModel
  from chains.llm import get_llm

  @pytest.fixture
  def mock_llm():
      return FakeListChatModel(responses=['{"company": "Acme", "role": "Eng", ...}'])
  ```

### File System
- Use `tmp_path` pytest fixture for tracker tests and output directory tests.
- Use ` monkeypatch` to override `config.RESUME_PATH` for resume parser tests.

### Ollama API
- Use `responses` library or `unittest.mock.patch("requests.get")` for `verify` command tests.

### PDF Parsing
- Mock `pdfplumber.open` to yield mock page objects with `.extract_text()` returning controlled strings.

   ```python
       return fake
   ```

## 5. Coverage Goals

| Module | Target | Rationale |
|--------|--------|-----------|
| `utils/resume_parser.py` | 90%+ | Core dependency; PDF parsing is fragile |
| `utils/tracker.py` | 90%+ | Data integrity; every function should be tested |
| `chains/analyzer.py` | 80%+ | `validate_job_description` is easy; `analyze_job` needs mocked LLM |
| `chains/tailorer.py` | 70%+ | Mostly prompt formatting; one happy-path test suffices |
| `chains/cover_letter.py` | 70%+ | Same rationale as tailorer |
| `chains/followup.py` | 70%+ | Simplest chain |
| `config.py` | 60%+ | Validation logic only |
| `main.py` | 50%+ | CLI integration tests are higher effort; test critical paths |
| `app.py` | 0% (smoke) | Streamlit UI is best tested manually |

**Minimum viable test suite (highest value, lowest effort):**
1. `test_resume_parser.py` — 4 tests
2. `test_tracker.py` — 8 tests
3. `test_analyzer.py` — 2 tests (validate + analyze with mock)
4. `test_cli.py` — 2 tests (verify + tracker --show)

This gives confidence in PDF parsing, data integrity, and CLI wiring with ~16 tests.

## 6. CI Recommendation

Create `.github/workflows/test.yml`:

```yaml
name: Tests

on:
  push:
    branches: [main, feature/*]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r job_search_tool/requirements.txt
      - run: pip install pytest pytest-mock
      - run: pytest tests/ -v -m "not ollama"
```

### Tests to skip in CI
- Any test marked with `@pytest.mark.ollama` that requires a real Ollama connection.
- No tests in the minimum viable suite require a real Ollama connection if LLM calls are mocked.
