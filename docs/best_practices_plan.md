## 3. Error Handling

### 3.1 `wb.save(path)` unprotected in tracker
- **Files:** `utils/tracker.py:149`, `utils/tracker.py:287`, `utils/tracker.py:339`, `utils/tracker.py:388`
- **Issue:** `wb.save(path)` can raise `PermissionError` if the tracker is open in Excel. The raw exception propagates to the caller, producing an ugly traceback.
- **Fix:** Wrap `wb.save(path)` in try/except and re-raise as a user-friendly exception with a hint: "Tracker file may be open in another program. Close it and try again."
- **Severity:** Major

### 3.2 LLM calls inside chains are unwrapped
- **Files:** `chains/analyzer.py:104`, `chains/tailorer.py:46`, `chains/cover_letter.py:44`, `chains/followup.py:42`
- **Issue:** `.invoke()` calls are not wrapped in try/except. Connection errors bubble up. This is **intentional** because the CLI and Streamlit both wrap them, but the chains themselves are not self-defensive.
- **Fix:** Document that callers must handle `ConnectionError`. Alternatively, add a thin decorator/wrapper. **Acceptable as-is for now.**
- **Severity:** Minor

### 3.3 No bare except or pass
- **Finding:** None found. All except blocks catch specific exceptions.

## 5. Security

### 5.1 Path traversal in `--file` option
- **File:** `main.py:55-60`
- **Issue:** `get_job_description` takes a `Path` from Typer and reads it directly. A user could pass `--file ../../../etc/passwd`. Since this is a local CLI tool running as the user, the risk is limited to self-harm, but we should still validate.
- **Fix:** Check that the resolved path is a file (not a directory) and optionally warn if it resolves outside the project root.
- **Severity:** Minor

### 5.2 URL fetch timeout
- **File:** `main.py:77`
- **Finding:** `requests.get(url, timeout=10)` — correct. **No issue.**

### 5.3 URL scheme validation
- **File:** `main.py:70-73`
- **Finding:** Rejects URLs not starting with `http://` or `https://`. **No issue.**

### 5.4 Resume content in error messages
- **Finding:** No exception handler prints the full resume or job description text. All error messages use short placeholders or mention the path, not the content.
- **Status:** Clean. No action needed.

### 5.5 Filename sanitization
- **File:** `utils/helpers.py:28`
- **Finding:** `make_slug` already strips path separators and unsafe characters. However, it does not strip Windows reserved names (CON, PRN, AUX) or trailing dots/spaces.
- **Fix:** Add Windows reserved name check and strip trailing dots/spaces.
- **Severity:** Minor

## 7. Streamlit Best Practices

### 7.1 `st.session_state` not used for analysis results
- **File:** `app.py`
- **Issue:** When the user clicks a button or interacts with the app, Streamlit reruns the entire script. Analysis results are recomputed or lost on subsequent interactions (e.g., editing the cover letter text area does not persist the analysis above it if the user clicks elsewhere).
- **Fix:** Store `analysis` and `tailored_resume` / `cover_letter` in `st.session_state` so they persist across reruns.
- **Severity:** Major

### 7.2 Expensive operations on every rerun
- **File:** `app.py`
- **Issue:** `_cached_resume_text` is cached, which is good. But the job description is not cached, and the analysis is not cached. Every rerun re-validates the text area and potentially re-runs analysis if `run_clicked` is still True.
- **Fix:** Use `st.session_state` to gate re-runs. Clear the state when the user changes the job description.
- **Severity:** Medium

### 7.3 Long-running chain calls wrapped in `st.spinner()`
- **Finding:** The analyze call is wrapped. The tailor and cover letter calls in Full Application mode are wrapped by `st.progress()` which also shows a spinner implicitly. **Acceptable.**

### 7.4 Error messages use `st.error()`
- **Finding:** All user-facing errors in `app.py` use `st.error()`. No raw `st.write()` for errors. **No issue.**
